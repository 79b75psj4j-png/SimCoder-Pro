# Backend Architecture Documentation\n\n## Overview\n\nThe SimCoder Pro backend is built with Node.js and Express, using PostgreSQL for persistent data storage and Redis for caching and real-time features.\n\n## Technology Stack\n\n- **Runtime**: Node.js 18+\n- **Framework**: Express.js\n- **Database**: PostgreSQL 15\n- **Cache/Real-time**: Redis 7\n- **Authentication**: JWT (JSON Web Tokens)\n- **ORM**: Sequelize or TypeORM\n- **API**: RESTful with WebSocket support\n- **Validation**: Joi/Zod\n- **Testing**: Jest, Supertest\n- **Logging**: Winston\n- **Documentation**: Swagger/OpenAPI\n\n## Project Structure\n\n```\nbackend/\nв”њв”Ђв”Ђ src/\nв”‚   в”њв”Ђв”Ђ config/\nв”‚   в”‚   в”њв”Ђв”Ђ database.js\nв”‚   в”‚   в”њв”Ђв”Ђ redis.js\nв”‚   в”‚   в”њв”Ђв”Ђ auth.js\nв”‚ specialty\n- Account creation date\n- Last login\n- Status (active, suspended)\n- Preferences (theme, notifications)\n\n### Chart Model\n- ID, user_id\n- Chart type (office visit, procedure, etc.)\n- Specialty\n- Medical documentation (full text)\n- Chart status (assigned, in_progress, completed, audited)\n- Assigned codes (JSON)\n- User's code assignment\n- Correct codes\n- Created at, completed at\n- Difficulty level\n- Time spent coding\n\n### Code Models (ICD-10, CPT, HCPCS)\n- Code value\n- Description, long description\n- Category, section, chapter\n- Active status\n- 7th character requirements (ICD-10)\n- RVU values (CPT)\n- Clinical examples\n- Guidelines notes\n- Related codes\n- Effective date, deletion date\n- Last updated\n\n### Performance Model\n- User_id, date\n- Charts coded (count)\n- Accuracy percentage\n- Audit score\n- Denial rate\n- Average coding time\n- Charts completed today\n- Charts remaining\n- Performance rating\n\n### Audit Model\n- ID, user_id, chart_id\n- Audit date\n- Audit result (pass/fail)\n- Error details (JSON)\n- Error types (wrong diagnosis, wrong CPT, missing codes, etc.)\n- Suggested corrections\n- Auditor notes\n- Learning resources recommended\n\n### Training Model\n- ID, module_id\n- Title, description\n- Content type (video, lesson, quiz, case study)\n- Difficulty level\n- Estimated duration\n- Learning objectives\n- Content\n- Prerequisites\n- Related codes\n\n### UserProgress Model\n- User_id, training_id\n- Status (not_started, in_progress, completed)\n- Completion date\n- Score\n- Time spent\n- Notes/bookmarks\n\n## Core Services\n\n### Code Service\n- Search codes (exact, partial, description)\n- Get code details with guidelines\n- Validate code combinations\n- Check NCCI edits\n- Get bundling rules\n- Suggest codes based on documentation\n- Validate 7th characters\n\n### Chart Service\n- Generate random charts\n- Get user's chart queue\n- Assign chart to user\n- Submit coded chart\n- Retrieve chart for review\n- Calculate accuracy\n- Track time spent\n\n### Performance Service\n- Calculate daily metrics\n- Generate performance reports\n- Track accuracy trends\n- Calculate denial rates\n- Generate productivity reports\n- Monthly performance summary\n- Career advancement eligibility\n\n### Audit Service\n- Perform random audits\n- Score chart coding accuracy\n- Identify error types\n- Generate audit report\n- Suggest learning resources\n- Track audit history\n- Generate compliance reports\n\n### Training Service\n- Get training modules\n- Track progress\n- Generate learning paths\n- Get recommendations\n- Track completion\n- Score assessments\n- Generate certificates\n\n## API Endpoints (Key)\n\n### Authentication\n```\nPOST /api/auth/register - Register new user\nPOST /api/auth/login - Login user\nPOST /api/auth/refresh - Refresh JWT token\nPOST /api/auth/logout - Logout user\n```\n\n### User Management\n```\nGET /api/users/profile - Get user profile\nPUT /api/users/profile - Update profile\nGET /api/users/career - Get career progression\nPUT /api/users/specialty - Update specialty\n```\n\n### Charts/Coding\n```\nGET /api/charts/queue - Get chart queue\nGET /api/charts/:id - Get chart details\nPOST /api/charts/:id/submit - Submit coded chart\nGET /api/charts/:id/feedback - Get coding feedback\nGET /api/charts/history - Get user's chart history\n```\n\n### Code Reference\n```\nGET /api/codes/search - Search codes\nGET /api/codes/:type/:code - Get code details\nGET /api/codes/:type/:code/related - Get related codes\nGET /api/codes/:type/:code/bundling - Get bundling rules\nGET /api/codes/:type/:code/validation - Validate code\n```\n\n### Training\n```\nGET /api/training/modules - Get training modules\nGET /api/training/modules/:id - Get module details\nPOST /api/training/modules/:id/start - Start training\nPOST /api/training/modules/:id/complete - Complete training\nGET /api/training/progress - Get user progress\nGET /api/training/recommendations - Get learning recommendations\n```\n\n### Performance\n```\nGET /api/performance/daily - Get daily metrics\nGET /api/performance/monthly - Get monthly report\nGET /api/performance/trends - Get trends\nGET /api/performance/accuracy - Get accuracy details\nGET /api/performance/audit - Get audit results\n```\n\n### Audits\n```\nGET /api/audits - Get user audits\nGET /api/audits/:id - Get audit details\nPOST /api/audits/trigger - Trigger random audit\nGET /api/audits/:id/feedback - Get audit feedback\n```\n\n## Middleware\n\n### Authentication Middleware\n- Verify JWT token\n- Check user session\n- Validate permissions\n- Attach user to request\n\n### Error Handler\n- Catch all errors\n- Format error responses\n- Log errors\n- Send appropriate status codes\n\n### Validation Middleware\n- Validate request body\n- Validate query parameters\n- Validate URL parameters\n- Return validation errors\n\n### Rate Limiting\n- Prevent abuse\n- Limit requests per user\n- Implement sliding window\n- Return 429 on limit exceeded\n\n## Real-Time Features (WebSocket)\n\n- Live chart queue updates\n- Real-time performance updates\n- Notification system\n- Multi-user coding sessions\n- Supervisor notifications\n\n## Caching Strategy\n\n### Redis Keys\n- `codes:icd10:*` - ICD-10 codes\n- `codes:cpt:*` - CPT codes\n- `codes:hcpcs:*` - HCPCS codes\n- `modifiers:*` - Modifier information\n- `user:*:performance` - User performance metrics\n- `charts:queue:*` - User chart queues\n- `training:*` - Training content\n\n### Cache Expiration\n- Codes: 24 hours (rarely change during day)\n- Performance: 1 hour (calculated daily)\n- Training: 24 hours\n- User sessions: 24 hours\n\n## Security\n\n- Bcrypt password hashing\n- JWT token expiration (24h)\n- CORS configuration\n- Rate limiting\n- Input validation and sanitization\n- SQL injection prevention (ORM)\n- HTTPS enforcement\n- Environment variable protection\n\n## Logging\n\n- Winston logger\n- Log levels: error, warn, info, debug\n- File rotation\n- Console output in development\n- Structured logging with metadata\n- Request/response logging\n\n## Testing\n\n- Unit tests for services\n- Integration tests for API endpoints\n- Mock data and fixtures\n- Test database\n- Coverage reporting\n\n## Deployment\n\n- Docker containerization\n- Environment configuration\n- Database migrations\n- Health check endpoints\n- Graceful shutdown\n\n## Environment Variables\n\n```\nNODE_ENV=development\nAPI_PORT=5000\nDATABASE_URL=postgresql://user:password@host:5432/db\nREDIS_URL=redis://localhost:6379\nJWT_SECRET=your-secret-key\nJWT_EXPIRY=24h\n```\n"
