# API Documentation\n\n## Base URL\n\n```\nProduction: https://api.simcoderpro.com/api/v1\nDevelopment: http://localhost:5000/api/v1\n```\n\n## Authentication\n\nAll endpoints (except `/auth/login` and `/auth/register`) require a valid JWT token in the `Authorization` header:\n\n```\nAuthorization: Bearer <JWT_TOKEN>\n```\n\n## Response Format\n\nAll responses follow a standard format:\n\n```json\n{\n  \"success\": true,\n  \"data\": {},\n  \"message\": \"Success message\",\n  \"timestamp\": \"2024-01-15T10:30:00Z\"\n}\n```\n\nError responses:\n\n```json\n{\n  \"success\": false,\n  \"error\": \"ERROR_CODE\",\n  \"message\": \"Error description\",\n  \"details\": {},\n  \"timestamp\": \"2024-01-15T10:30:00Z\"\n}\n```\n\n## HTTP Status Codes\n\n- `200 OK`: Successful request\n- `201 Created`: Resource created\n- `400 Bad Request`: Invalid request parameters\n- `401 Unauthorized`: Missing or invalid authentication\n- `403 Forbidden`: Insufficient permissions\n- `404 Not Found`: Resource not found\n- `409 Conflict`: Resource already exists\n- `429 Too Many Requests`: Rate limit exceeded\n- `500 Internal Server Error`: Server error\n\n## Rate Limiting\n\n- **Public endpoints**: 100 requests per hour per IP\n- **Authenticated endpoints**: 1000 requests per hour per user\n- **Code search**: 500 requests per hour per user\n\nHeaders:\n- `X-RateLimit-Limit`: Total requests allowed\n- `X-RateLimit-Remaining`: Remaining requests\n- `X-RateLimit-Reset`: Timestamp when limit resets\n\n## Endpoints\n\n### Authentication\n\n####
